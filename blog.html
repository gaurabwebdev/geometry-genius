The DOM (Document Object Model) is a programming interface for web documents which represents the page so that programs can change the document structure, style, and content. It represents the HTML orXML documents as nodes and objects; that way, programming languages can interact with the page.
The purposes of DOM:
  • We can navigate the documents, access individual elements, and modify their content and attributes using the tree-like-structure provided by DOM.
  • Without reloading the retired webpage, DOM allows web developers to create scripts that can modify the content and structure of a web page dynamically.
  • Scripts can interact with web page content, styles, and behavior also can manipulate them by DOM. This feature is used for creating interactive web applications and user interfaces.
Finally we can say, DOM plays a crucial role for creating such dynamic, interactive, and accessible web content.

There are many ways or methods to get or select HTML elements in DOM. Most commonly used are mentioned below:
  • getElementById() - method
  • getElementById() - method
  • getElementsByClassName() - method
  • getElementsByTagName() - method
  • querySelector() - method
  • querySelectorAll() - method

Event bubbling is a type of DOM event propagation where the event first triggers on the innermost target element, and then successively triggers on the ancestors of the target element in the same nesting hierarchy till it reaches the outermost DOM element or document object.

A function passed into another function as an argument is callback function. Its invoked inside the outer function to complete some kind of routine or action.

The purpose of callback functions is to provide ways to execute codes in a non-blocking and asynchronous manner because it's important for the performance and responsiveness in many situations such as event handling, asynchronous programming, iteration, modularity, error handling etc.
